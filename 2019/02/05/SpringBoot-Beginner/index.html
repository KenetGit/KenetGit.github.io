SpringBoot基础知识

发表于 2019-02-05

首先需要明确对SpringBoot的认识，SpringBoot并不是什么新框架，而是作为Spring应用开发的一个快速孵化器，使用SpringBoot可以减除开发Spring应用中的很多手动配置。SpringBoot帮助开发者自动配置了应用的基本配置。

SpringBoot项目工程的创建过程比较简单，对比原先使用Spring框架，本文记录一些SpringBoot与Spring的异同点，帮助从Spring开发过渡到SpringBoot开发。如果没有使用过SpringBoot，最好自己找个教程手动创建一遍spingboot项目工程，运行起来试试。

创建完项目工程后点击项目的extend看看是不是导入了很多jar包，这就是SpringBoot的作用之处。

使用SpringBoot的好处

其实就是简单、快速、方便！平时如果我们需要搭建一个spring web项目的时候需要怎么做呢？

1）配置web.xml，加载spring和spring mvc

2）配置数据库连接、配置spring事务

3）配置加载配置文件的读取，开启注解

4）配置日志文件

….

配置完成之后部署tomcat 调试， springboot就是简化这些过程。

认识SpringBoot的启动器

罗列了一些简单开发中会经常用到的启动器。

  • spring-boot-starter
    这是Spring Boot的核心启动器，包含了自动配置、日志和YAML。
  • spring-boot-starter-web
    支持全栈式Web开发，包括Tomcat和spring-webmvc。

  • spring-boot-starter-test
    支持常规的测试依赖，包括JUnit、Hamcrest、Mockito以及spring-test模块。

  • spring-boot-starter-thymeleaf
    支持Thymeleaf模板引擎，包括与Spring的集成。

  • spring-boot-starter-aop
    支持面向方面的编程即AOP，包括spring-aop和AspectJ。

  • spring-boot-starter-velocity
    支持Velocity模板引擎。

  • spring-boot-starter-jdbc
    支持JDBC数据库。

  • spring-boot-starter-jersey
    支持Jersey RESTful Web服务框架。

  • spring-boot-starter-websocket
    支持WebSocket开发。

  • spring-boot-starter-actuator
    帮助监控和管理应用。

  • spring-boot-starter-cache
    支持Spring的Cache抽象。

  • spring-boot-starter-data-gemfire
    支持GemFire分布式数据存储，包括spring-data-gemfire。

  • spring-boot-starter-data-jpa
    支持JPA（Java Persistence API），包括spring-data-jpa、spring-orm、hibernate。

  • spring-boot-starter-data-rest
    通过spring-data-rest-webmvc，支持通过REST暴露Spring Data数据仓库。

  • spring-boot-starter-data-solr
    支持Apache Solr搜索平台，包括spring-data-solr。

  • spring-boot-starter-freemarker
    支持FreeMarker模板引擎。

  • spring-boot-starter-hateoas
    通过spring-hateoas支持基于HATEOAS的RESTful Web服务。

  • spring-boot-starter-jetty
    引入了Jetty HTTP引擎（用于替换Tomcat）
  • pring-boot-starter-log4j
    支持Log4J日志框架。
  • spring-boot-starter-logging
    引入了Spring Boot默认的日志框架Logback。
  • spring-boot-starter-tomcat
    引入了Spring Boot默认的HTTP引擎Tomcat。

举例来说，一个简单的SpringBoot Web应用，应该会包含以下启动器。

可以看到，在Spring开发中，我们通过在pom.xml导入一系列包依赖，而springboot中导入一系列启动器，实际上启动器就是整合、管理了包依赖。

认识SpringBoot的项目结构

这个一个简单的web应用的项目结构，我一般都是如此安排各种文件的存放位置。

  • 应用启动入口

只需运行这个main方法就可以启动springboot应用，比起spring要快捷得多。

  • springboot配置文件

采用YAML格式。相关知识应了解YAML的语法以及springboot配置中各种属性如何用yaml格式表示

  • 注解扫描

springboot自动开启注解扫描，因此不需要配置。将controller类，service类等予以对应的注解即可被sprngboot扫描。
相对于spring，springboot也引入了一些新的注解，之后在项目应用中会逐一介绍。

开发入门

了解了SpringBoot的项目结构后，其他开发工作与Spring开发并没有多大不同。
