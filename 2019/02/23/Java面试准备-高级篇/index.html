JDBC

JDBC是一套面向对象的应用程序接口，制定了统一的访问各类关系型数据库的标准接口，为各个数据库厂商实现JDBC驱动提供了接口标准。简单说，原先我们使用SQL语句都是通过控制台输入，而JDBC让我们可以用Java向数据库发送SQL语句。

JDBC的驱动：各个数据库厂家提供的遵从JDBC接口规范而开发实现的数据库操作，比如，mysql提供的JDBC驱动，让我们可以通过Java操作MySql数据库。说明白一点，JDBC是接口，而具体的驱动则是接口的实现。

java.sql.Driver是一个接口，mysql驱动中的java.sql.Driver接口的实现类为com.mysql.jdbc.Driver

JDBC的核心类和接口有：DriverManager，Connection、Statement，和ResultSet。

通常使用JDBC完成以下操作：同数据库建立连接；向数据库发送SQL语句；处理从数据库返回的结果。

使用原生的JBDC时，我们一般都会对其进行再次的封装，减少重复的代码，比如，连接数据库，释放连接等，而后来兴起的ORM，即面向关系映射的数据库操作使得开发者不需要再编写JDBC的代码，但是不管第三方类库提供了如何便捷的封装，JDBC始终是面向数据库操作的基石，是封装的底层知识。

PreparedStatement与Statement

  • PreparedStatement可以写动态参数化的查询
  • SQL语句会预编译在数据库系统中。执行计划同样会被缓存起来，当有多次请求时，因为第一次请求已经预编译并且缓存起来，以后的查询操作就相对要快。
  • PreparedStatement可以防止SQL注入式攻击

连接池

同线程池相比可想而知，数据库连接池就是将多个连接放入池中，每个给一个连接请求分配一个连接，用完之后归还到连接池中。因为像数据库连接建立和关闭都比较消耗时间，特别是当连接请求频繁时会增加服务器的资源压力。

Servlet

Servlet是用来处理客户端请求并产生动态网页内容的java类，是基于java语言的web服务端的编程技术，它能处理http请求并产生http响应。在开发中不会单独使用Servlet，因为SpringMVC基于Servlet技术做了很好的改进。

所有的Servlet都必须实现的核心接口javax.servlet.Servlet，每一个Servlet都需要直接或间接实现这个接口。Servlet使用多线程，可以为每个请求开启一个线程，并行处理多个请求服务。

  • Servlet的生命周期
  1. 对每一个客户端的请求，Servlet引擎载入Servlet，调用它的init()方法完成初始化；
  2. Servlet对象通过为每一个请求单独调用service()方法来处理客户端的请求
  3. 最后，调用Servlet的destroy方法回收Servlet对象。

  • Servlet链

    Servelt链是指一个Servlet将它的输出传递给另一个Servlet，一次来推，最后一个Servlet负责把响应发送给客户端。

重定向和转发

sendRedict是浏览器方面的跳转，是由服务端告诉客户端去做这样一件事；forward是在服务器内部的跳转，将请求传递给其他request handler。重定向总共需要客户端发起2次请求，而转发是同一次请求。重定向后，之前的请求作用域范围就失效了，而转发则是保留的，因为它还是在同一次请求中。重定向一般会比转发要慢，所以应该尽量避免重定向。
