集合类的特点

ArrayList

对ArrayList的特性概述是，“动态增长和缩减的索引序列“，是基于扩展的数组实现的。

ArrayList内部有3个数组，一个是静态的

可见，静态(1)和（2)在多个空实例的情况下可以减少无用的开销，因为这些实例的elementData会指向同一个方法区中的静态数组。

在第（2）种情况下，等到开始插入第一个元素的时候，ArrayList对象的elementData才会真正开始计算需要的实际数组大小。（在不大于默认值的情况下是默认值，超过默认值则开始扩容）

常量池位于方法区，方法区位于堆内存。

那ArrayList是如何实现动态性的呢？以及动态扩充后，那些空闲的元素位置如何被对待？

看看ArrayList的元素存放在哪里，elementData是关键字transient修饰的。

transient的作用是什么呢？

transient用来表示一个域不是该对象序行化的一部分，当一个对象被序行化的时候，transient修饰的变量的值是不包括在序行化的表示中。如是这样的话，序列化ArrayList的时候，其内容岂不是被忽略了？其实ArrayList是这样处理的。

ArrayList在序列化的时候会调用writeObject，直接将size和element写入ObjectOutputStream；反序列化时调用readObject，从ObjectInputStream获取size和element，再恢复到elementData。

为什么不直接用elementData来序列化，而采用上诉的方式来实现序列化呢？原因在于elementData是一个缓存数组，它通常会预留一些容量，等容量不足时再扩充容量，那么有些空间可能就没有实际存放元素，采用上诉的方式来实现序列化时，就可以保证只序列化那些实际存放的元素，而不是整个数组，从而节省空间和时间。

    优点：ArrayList 底层以数组实现，是一种随机访问模式。ArrayList 实现了 RandomAccess 接口，因此查找的时候非常快。ArrayList 在顺序添加一个元素的时候非常方便。

    缺点：删除元素的时候，需要做一次元素复制操作。如果要复制的元素很多，那么就会比较耗费性能。插入元素的时候，也需要做一次元素复制操作，缺点同上。

    ArrayList比较适合顺序添加、随机访问的场景。

Vector

和ArrayList相比，二者都实现了AbstractList，但是，Vector类的所有方法都是同步的，可以由两个线程安全地访问同一个Vector对象，维护同步会耗费时间。ArrayList方法不是同步的，因此，在不需要同步的时候使用ArrayList。由源码可见，Vector的方法都使用了synchronized修饰。

LinkedList

LinkedList是可以在任何位置进行高效插入和删除操作的有序序列（链表），而它的内部正是基于链表实现的。并且记录了链表的头结点和尾结点，提供了一系列方法用于在指定位置插入和删除。

遍历一个 List 有哪些不同的方式？每种方法的实现原理是什么？Java 中 List 遍历的最佳实践是什么？

答案：遍历方式有以下几种：

  1. for 循环遍历，基于计数器。在集合外部维护一个计数器，然后依次读取每一个位置的元素，当读取到最后一个元素后停止。
  2. 迭代器遍历，Iterator。Iterator 是面向对象的一个设计模式，目的是屏蔽不同数据集合的特点，统一遍历集合的接口。Java 在 Collections 中支持了 Iterator 模式。
  3. foreach 循环遍历。foreach 内部也是采用了 Iterator 的方式实现，使用时不需要显式声明 Iterator 或计数器。优点是代码简洁，不易出错；缺点是只能做简单的遍历，不能在遍历过程中操作数据集合，例如删除、替换。

最佳实践：Java Collections 框架中提供了一个 RandomAccess 接口，用来标记 List 实现是否支持 Random Access。

  • 如果一个数据集合实现了该接口，就意味着它支持 Random Access，按位置读取元素的平均时间复杂度为 O(1)，如ArrayList。
  • 如果没有实现该接口，表示不支持 Random Access，如LinkedList。

推荐的做法就是，支持 Random Access 的列表可用 for 循环遍历，否则建议用 Iterator 或 foreach 遍历。

Hashset

HashSet是没有重复元素的无序集合，也就是数学上一般的集合概念，无序、互异。它的内部使用的是HashMap来存放元素，通过一些方法来保证插入的元素不会重复。

LinkedHashSet

LinkedHashSet是可以记录插入顺序的集合，同时它也是一个非线程安全的集合类。

LinkedHashSet 链接的散列集合, 实际上就是将散列的项用双向链表链接起来，并且记录项的插入顺序。

TreeSet

TreeSet是有序集合，实际上是基于红黑树实现的，将元素添加到树中比添加到散列表中要慢一些，但是删除的速度要快。

选择HashSet还是TreeSet？
如果不需要排序就不必付出排序的开销，此时选择HashSet。

HashMap

特点

HashMap 基于Map接口实现，存储键值对时，可以接收 null 为键值。HashMap 是非同步的。

工作原理

HashMap虽然使用链表解决哈希冲突，但是与C++相比,并不是直接通过hash(key)得到桶号，然后将value存储到桶所对应的链表。HashMap使用内部静态类Node作为链表的结点，这意味着不仅仅是存储value，而是同时存储hash，key，还有value。（可是，对于同一个桶号所在的链表上的结点的hash值不是一样的吗？为什么还需要为每个结点存储一个hash值，这样做岂不是浪费了存储空间？）

判断结点“相等“的方法，可见只有当key和value同时相等才会被判定为true。

  • 调用 put 方法时，使用键值对中的Key对象的 hashCode() 和哈希算法找出存储键值对的索引（桶号）。
    键值对 Entry 存储在 LinkedList 中，如果存在 Entry，使用 equals() 方法来检查 Key 是否已经存在：

    第一次put的时候才会去初始化hashmap，所以是初始化是延迟的。

注意，使用 ==和equals二者进行比较的区别，前者是直接比较地址，后者需要具体结合该类是否重写了equals方法。

  • 调用 get 方法时，HashMap 使用键值 Key hashCode() 来找到数组中的索引，然后使用 equals() 方法找出正确的 Entry，返回 Entry 中的 Value。
能否使用任何类作为 Map 的 key？

回答：可以使用任何类作为 Map 的 key，然而在使用之前，需要考虑以下几点：

  • 如果类重写了 equals() 方法，也应该重写 hashCode() 方法。
  • 类的所有实例需要遵循与 equals() 和 hashCode() 相关的规则。
  • 如果一个类没有使用 equals()，不应该在 hashCode() 中使用它。
  • 用户自定义 Key 类最佳实践是使之为不可变的，这样 hashCode() 值可以被缓存起来，拥有更好的性能。不可变的类也可以确保 hashCode() 和 equals() 在未来不会改变，这样就会解决与可变相关的问题了。

HashMap是支持null键和null值的，而HashTable在遇到null时，会抛出NullPointerException异常。这并不是因为HashTable有什么特殊的实现层面的原因导致不能支持null键和null值，这仅仅是因为HashMap在实现时对null做了特殊处理，将null的hashCode值定为了0，从而将其存放在哈希表的第0个bucket中。

HashMap线程不安全的体现（重中之重）

（1）链表出现环

（2）部分数据丢失

  • 顺便再记录一个HashMap的问题：为什么String, Interger这样的wrapper类适合作为键？

    String, Interger这样的wrapper类作为HashMap的键是再适合不过了，而且String最为常用。因为String是不可变的，也是final的，而且已经重写了equals()和hashCode()方法了。其他的wrapper类也有这个特点。不可变性是必要的，因为为了要计算hashCode()，就要防止键值改变，如果键值在放入时和获取时返回不同的hashcode的话，那么就不能从HashMap中找到你想要的对象。

    不可变性还有其他的优点如线程安全。如果你可以仅仅通过将某个field声明成final就能保证hashCode是不变的，那么请这么做吧。因为获取对象的时候要用到equals()和hashCode()方法，那么键对象正确的重写这两个方法是非常重要的。如果两个不相等的对象返回不同的hashcode的话，那么碰撞的几率就会小些，这样就能提高HashMap的性能。

  • fail-fast

    如果在使用迭代器的过程中有其他线程修改了map，那么将抛出ConcurrentModificationException，这就是所谓fail-fast策略。

    如果对同步性没有要求，应该使用HashMap;如果需要并发访问，则应该使用ConcurrentHashMap

WeakHashMap

  • 当对键的唯一引用来自散列条目时，这一数据结构将与垃圾回收器协同工作一起删除键/值对
  • java.util.WeakHashMap 使用 弱引用（WeakReference 对象）保存键。
  • WeakReference对象将引用保存到另外一个对象中，如果内存中只剩下WeakReference对象对某个键值对的引用，那么将这个WeakReference对象放入待回收的队列中，WeakHashMap周期性检查队列，以找出新添加的WeakReference对象。
