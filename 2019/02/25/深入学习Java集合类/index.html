深入学习Java集合类

发表于 2019-02-25

直奔主题，先来看看Java 集合类的框架结构。下图中所示的，是一个集合接口的继承关系图。

可见。集合框架中最基本的接口有Iterable、Collection、Map、Iterator。

集合类型 描述
ArrayList 可以动态增长和缩减的索引序列
LinkedList 可以在任何位置进行高效插入和删除操作的有序序列
HashSet 没有重复元素的无序集合
LinkedHashSet 一种可以记住元素插入顺序的集合
EnumSet 一种包含枚举类型值的集合
TreeSet 一种有序集合
PriorityQueue 一种允许高效删除最小元素的队列（优先级队列，由堆结构实现）
ArrayDeque 一种用循环数组实现的双端队列
HashMap 一种存储键值对的映射表
LinkedHashMap 一种可以记住键值对添加顺序的映射表
TreeMap 一种键值对有序排列的映射表
EnumMap 键值对书友枚举类型的映射表
WeakHashMap 当该映射表引用个数为0时可以被GC回收的映射表（弱引用的哈希表）
IndentityHashMap 用 == 而不是equals比较键值的哈希表

集合类的特点

ArrayList

对ArrayList的特性概述是，“动态增长和缩减的索引序列“，那ArrayList是如何实现动态性的呢？以及动态扩充后，那些空闲的元素位置如何被对待？

看看ArrayList的元素存放在哪里，elementData是关键字transient修饰的。

transient的作用是什么呢？

transient用来表示一个域不是该对象序行化的一部分，当一个对象被序行化的时候，transient修饰的变量的值是不包括在序行化的表示中。如是这样的话，序列化ArrayList的时候，其内容岂不是被忽略了？其实ArrayList是这样处理的。

ArrayList在序列化的时候会调用writeObject，直接将size和element写入ObjectOutputStream；反序列化时调用readObject，从ObjectInputStream获取size和element，再恢复到elementData。

为什么不直接用elementData来序列化，而采用上诉的方式来实现序列化呢？原因在于elementData是一个缓存数组，它通常会预留一些容量，等容量不足时再扩充容量，那么有些空间可能就没有实际存放元素，采用上诉的方式来实现序列化时，就可以保证只序列化那些实际存放的元素，而不是整个数组，从而节省空间和时间。

Vector

和ArrayList相比，二者都实现了AbstractList，但是，Vector类的所有方法都是同步的，可以由两个线程安全地访问同一个Vector对象，维护同步会耗费时间。ArrayList方法不是同步的，因此，在不需要同步的时候使用ArrayList。由源码可见，Vector的方法都使用了synchronized修饰。

LinkedList

LinkedList是可以在任何位置进行高效插入和删除操作的有序序列（链表），而它的内部正是基于链表实现的。并且记录了链表的头结点和尾结点，提供了一系列方法用于在指定位置插入和删除。

指定位置插入链表。

所以，如果要高效地插入和删除元素，选择LinkedList比ArrayList更合适。

Hashset

HashSet是没有重复元素的无序集合，也就是数学上一般的集合概念，无序、互异。它的内部使用的是HashMap来存放元素，通过一些方法来保证插入的元素不会重复。

LinkedHashSet

LinkedHashSet是可以记录插入顺序的集合，同时它也是一个非线程安全的集合类。

LinkedHashSet 链接的散列集合, 实际上就是将散列的项用双向链表链接起来，并且记录项的插入顺序。

TreeSet

TreeSet是有序集合，实际上是基于红黑树实现的，将元素添加到树中比添加到散列表中要慢一些，但是删除的速度要快。

选择HashSet还是TreeSet？
如果不需要排序就不必付出排序的开销，此时选择HashSet。

HashMap

如果对同步性没有要求，应该使用HashMap;如果需要并发访问，则应该使用ConcurrentHashMap

WeakHashMap

  • 当对键的唯一引用来自散列条目时，这一数据结构将与垃圾回收器协同工作一起删除键/值对
  • java.util.WeakHashMap 使用 弱引用（WeakReference 对象）保存键。
  • WeakReference对象将引用保存到另外一个对象中，如果内存中只剩下WeakReference对象对某个键值对的引用，那么将这个WeakReference对象放入待回收的队列中，WeakHashMap周期性检查队列，以找出新添加的WeakReference对象。
